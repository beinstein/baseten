If you can’t connect to PostgreSQL

There are a number of things that can prevent BaseTen Assistant from connecting to your PostgreSQL database. This check list assumes you have administrator access to the database server and host.

Check list:

The connection details may not be correct. Check that you have a valid user name and password, and that they have been granted access to the database.

See PostgreSQL 8.3 documentation on GRANT

Check that the database server is accessible. If you used Bonjour to locate the database, or the database is on this machine you can skip this step. Use the Network Utility, or the Terminal to ping the database host to verify.

Launch Network Utility
Launch Terminal

Check that firewall on the database host allows access.

See “Setting firewall access for services and applications” in Mac Help
Launch Security Preferences

Check that PostgreSQL’s Host Based Authentication is configured to allow access to the database from your machine and your user account.

See PostgreSQL 8.3 documentation on Client Authentication
